# Plan: UI/UX Modernization (2025-09-11)

## 1. Overview

This document outlines the phased plan to refactor and modernize the Knovy application's user interface. The goal is to create a more intuitive, consistent, and visually appealing user experience, while improving the underlying codebase for better maintainability and future development velocity.

**Lead Architect:** Gemini
**Stakeholders:** Product Team

---

## Phase 1: Codebase Refactoring and Component Consolidation

**Goal:** Establish a clean and consistent foundation for future UI work. This phase is non-visual but critical for long-term success.

**Status:** **Proposed. Awaiting Approval.**

### Task Analysis

- The current component structure in `@apps/app/src/renderer/src/components/main/` has redundant components and inconsistent naming conventions (e.g., `Popup` vs. `Window`).
- This leads to developer confusion and makes maintenance difficult.
- We will audit, prune, and standardize the components to create a single source of truth for core UI elements.

### Sub-Agent Assignments & Task Breakdown

1.  **Task 1.1: Component Audit**
    - **Description:** Analyze all components in `apps/app/src/renderer/src/components/main/` to identify redundancies and naming inconsistencies.
    - **AGENT:** `@agents/core/code-archaeologist`

2.  **Task 1.2: Consolidate Settings Components**
    - **Description:** Merge the functionality of `SettingsPopup.tsx` and `AdvancedSettingsWindow.tsx` into a single, new component: `SettingsModal.tsx`. This component will house all application settings.
    - **AGENT:** `@agents/specialized/react-component-architect`

3.  **Task 1.3: Consolidate Screen Preview Components**
    - **Description:** Merge `ScreenPreviewPopup.tsx` and `ScreenPreviewWindow.tsx` into a single `ScreenPreview.tsx`. The distinction between a "Popup" and a "Window" is unnecessary; a single component can handle the preview functionality.
    - **AGENT:** `@agents/specialized/react-component-architect`

4.  **Task 1.4: Rename Core Interaction Components**
    - **Description:** Rename `FeaturesPopup.tsx` to `ActionsPanel.tsx` to better reflect its new role as the primary AI interaction hub.
    - **AGENT:** `@agents/universal/frontend-developer`

5.  **Task 1.5: Refactor Main Bar Naming**
    - **Description:** Clarify the naming of the main control bar. Rename `HeaderBar.tsx` to `MainControlBar.tsx` and `MainBar.tsx` to `MainController.tsx` to distinguish between the UI view and its controller logic.
    - **AGENT:** `@agents/universal/frontend-developer`

6.  **Task 1.6: Prune Unused Components**
    - **Description:** After consolidation and renaming, safely remove the old, now-unused component files.
    - **AGENT:** `@agents/universal/frontend-developer`

7.  **Task 1.7: Update Imports & Verify**
    - **Description:** Update all import paths across the application to reflect the new component names and structure. Run the application and perform a smoke test to ensure no breaking changes were introduced.
    - **AGENT:** `@agents/universal/frontend-developer`

### Execution Order

- **Sequential**: Task 1.1 → (Task 1.2, Task 1.3) → Task 1.4 → Task 1.5 → Task 1.6 → Task 1.7
- Tasks 1.2 and 1.3 can be run in parallel.

## Phase 2: Layout and Core Panel Interaction

**Goal:** Implement the new dual-panel layout and establish the foundational behavior of the `ChatPanel` and the `ActionsPanel`.

**Status:** **Draft**

### Task Breakdown

1.  **Task 2.1: Implement Dual-Panel Layout Container**
    - **Description:** Create a new root-level component that uses Flexbox or CSS Grid to manage the layout of the `ActionsPanel` and `ChatPanel`.
    - **AGENT:** `@agents/specialized/react-component-architect`

2.  **Task 2.2: Implement Responsive Panel Logic**
    - **Description:** Add logic to the layout container to handle two states:
      - If both panels are visible, display them side-by-side with a `16px` gap.
      - If only one panel is visible, it should animate to the center-top of the window.
    - **AGENT:** `@agents/specialized/react-component-architect`

3.  **Task 2.3: Relocate Input to ActionsPanel**
    - **Description:** Move the text input form from `ChatPanel.tsx` to the bottom of `ActionsPanel.tsx`. `ChatPanel` will now be a display-only component for transcripts.
    - **AGENT:** `@agents/universal/frontend-developer`

4.  **Task 2.4: Create ActionsPanel Initial State**
    - **Description:** Design the initial state of the `ActionsPanel` to be a 2x2 grid of AI action buttons (Answer, Screenshot, File, etc.).
    - **AGENT:** `@agents/specialized/react-component-architect`

5.  **Task 2.5: Relocate "View History" Button**
    - **Description:** Remove the "View History" button from the `ActionsPanel` and add it as a prominent navigation item within `SettingsModal.tsx`.
    - **AGENT:** `@agents/universal/frontend-developer`

### Execution Order

- **Sequential**: Task 2.1 → Task 2.2 → (Task 2.3, Task 2.4) → Task 2.5
- Tasks 2.3 and 2.4 can be run in parallel.

## Phase 3: ActionsPanel UI Refinement and Interactivity

**Goal:** Transform the `ActionsPanel` from a static button grid into a dynamic conversational interface.

**Status:** **Draft**

### Task Breakdown

1.  **Task 3.1: Implement Contextual Visibility**
    - **Description:** Update the `MainControlBar.tsx` to only show the `ActionsPanel` toggle button when the application is in the "listening" state.
    - **AGENT:** `@agents/universal/frontend-developer`

2.  **Task 3.2: Implement Panel Transformation Animation**
    - **Description:** Using a library like `motion/react`, create a one-time animation for when an AI action is clicked. The grid of buttons should scale down and fade out, while simultaneously reappearing as a horizontal row of smaller icons above the input field.
    - **AGENT:** `@agents/specialized/react-component-architect`

3.  **Task 3.3: Implement Conversational View**
    - **Description:** The space previously occupied by the button grid should fade in to become the conversational view for displaying AI responses.
    - **AGENT:** `@agents/universal/frontend-developer`

4.  **Task 3.4: Integrate AI Responses and Loading States**
    - **Description:** Connect the `ActionsPanel` to the `useAIInteraction.ts` hook. Render AI responses in the new conversational view. Implement loading states (e.g., spinner, disabled inputs) while waiting for an AI response.
    - **AGENT:** `@agents/universal/frontend-developer`

### Execution Order

- **Sequential**: Task 3.1 → Task 3.2 → Task 3.3 → Task 3.4

---

## Phase 4: ChatPanel UI Refinement and Functionality

**Goal:** Enhance the `ChatPanel` with summarization features and improved usability.

**Status:** **Draft**

### Task Breakdown

1.  **Task 4.1: Implement Tabbed Interface**
    - **Description:** Add a tab component to the top of `ChatPanel.tsx` with two tabs: "Transcription" (default) and "Summary".
    - **AGENT:** `@agents/specialized/react-component-architect`

2.  **Task 4.2: Enable Auto-Scrolling**
    - **Description:** Ensure the "Transcription" tab content automatically scrolls to the bottom as new transcriptions arrive. If user scrolls up to review past content, the auto-scrolling should be temporarily disabled until they scroll back down.
    - **AGENT:** `@agents/universal/frontend-developer`

3.  **Task 4.3: Implement Initial Summarization**
    - **Description:** Connect the "Summary" tab to the `sendContextToAI('summary')` function in `useAIInteraction.ts`. Trigger this on the first click of the tab and display a loading state.
    - **AGENT:** `@agents/universal/frontend-developer`

4.  **Task 4.4: Implement Continuous Summarization**
    - **Description:** Implement a mechanism to periodically update the summary. Create a configurable variable `SUMMARY_TIME_WINDOW_SECONDS` (defaulting to 60) and use it to summarize new transcriptions since the last summary.
    - **AGENT:** `@agents/universal/frontend-developer`

### Execution Order

- **Sequential**: Task 4.1 → Task 4.2 → Task 4.3 → Task 4.4
